# База знаний — DM Task Manager

## Деплой на Render

**Проблема:** Изменения видны на localhost, но НЕ видны на https://dm-task-manager.onrender.com/

**Причина:** Render деплоит из Git-репозитория. Пока изменения не запушены — Render показывает старую версию.

**Решение:**
```bash
cd ~/Desktop/dm-task-manager
git add -A
git commit -m "описание изменений"
git push
```
После пуша Render автоматически подхватит и задеплоит (1-2 мин).

**ВАЖНО: ВСЕГДА ДЕЛАТЬ ПУШ ПОСЛЕ ИЗМЕНЕНИЙ!**
Без `git push` изменения видны ТОЛЬКО на localhost. На Render ничего не обновится.

**Порядок работы (ОБЯЗАТЕЛЬНЫЙ):**
1. Внести изменения в код
2. Проверить на localhost:8080
3. **ОБЯЗАТЕЛЬНО:** `git add -A && git commit -m "..." && git push` — деплой на Render
4. Подождать 1-2 минуты
5. Проверить на https://dm-task-manager.onrender.com/

**Быстрая команда (копируй и вставляй):**
```bash
cd ~/Desktop/dm-task-manager && git add -A && git commit -m "update" && git push
```

## ⚠️ ОБЯЗАТЕЛЬНОЕ ПРАВИЛО: ОБУЧЕНИЕ ПРИ КАЖДОМ ИЗМЕНЕНИИ

**ВАЖНО:** С ЛЮБЫМ изменением интерфейса, добавлением фичи или удалением старой —
ОБЯЗАТЕЛЬНО добавлять/обновлять интерактивное обучение. Всегда супер-подробное.

**Что это значит:**
1. Любая новая вкладка/раздел → добавить туториал в `getQuickStepsForPage(page)` и `getDetailedSteps(page)` в `templates/index.html`
2. Любая новая кнопка/взаимодействие → пошагово объяснить в обучении
3. Любая удалённая фича → убрать упоминания из старых туториалов и добавить в changelog
4. После добавления шагов обучения — **автоматически запускать его при первом входе на страницу**,
   чтобы пользователь сразу увидел новый функционал без необходимости искать кнопку «?»
5. Шаги должны быть **супер-детальными**: объяснять не только «куда нажать», но и «зачем» и «что произойдёт»

**Шаблон шага обучения:**
```javascript
{
    target: 'СЕЛЕКТОР_ЭЛЕМЕНТА',    // CSS-селектор
    title: '🎯 Короткий заголовок',
    text: 'Супер-подробное объяснение: что это, зачем, какой результат.',
    page: 'kanban',                  // на какой странице
    position: 'right'                // где показывать подсказку
}
```

**Флаг «уже показано»:** хранить в localStorage с ключом вида `dm_onboarding_<feature>_v<version>`,
чтобы при добавлении новой версии обучения оно показывалось ВСЕМ пользователям снова, даже если
они уже видели предыдущую версию.

**Где искать существующие туториалы:**
- `startPageTut(page, mode)` в `templates/index.html` — запуск туториала
- `getQuickStepsForPage(page)` — быстрые шаги
- `getDetailedSteps(page)` — подробные шаги (всегда использовать их по умолчанию)
- `_tutBuildSteps(pages, mode)` — сбор шагов для нескольких страниц
- `_tutShow()` — движок отрисовки шагов (spotlight, card, arrow)
- `_tutCreateSpotlight(targetEl, allowClickThrough)` — spotlight-оверлей
- `_tutDrawArrow(targetEl, cardEl)` — стрелка от карточки к элементу

### Дизайн-правила обучения (ОБЯЗАТЕЛЬНЫЕ)

1. **Первый шаг — приветственный на весь экран.** Флаг `_welcome: true` ставится автоматически
   на первый info-шаг в `_tutBuildSteps()`. Показывает заголовок, описание и список «Что вы узнаете».

2. **Размер карточки** — минимум 520px ширина, заголовок 21px, текст 16px, line-height 1.7.
   Карточка НЕ должна быть маленькой или с мелким текстом.

3. **Интерактивные шаги** — если шаг требует действия от пользователя, добавить поле `action`:
   ```javascript
   { icon:'📊', title:'Статистика', description:'...', selector:'#statsGrid',
     action:'Нажмите на любую плитку, чтобы отфильтровать задачи' }
   ```
   Подсвеченный элемент становится кликабельным (`allowClickThrough: true`).
   Кнопка «Дальше» заменяется на «Пропустить действие →».

4. **Подсказка действия** — жёлтая плашка с `👆` и жирным текстом. Хорошо читаемая, контрастная.

5. **Стрелка к элементу** — SVG-стрелка с pulse-анимацией (`tutArrowPulse`) указывает от карточки к
   подсвеченному элементу. Пользователь всегда видит КУДА нажимать.

6. **Spotlight** — `box-shadow: 0 0 0 9999px rgba(15,23,42,0.6)` + `border: 3px solid rgba(99,102,241,0.7)`.
   Для интерактивных шагов spotlight пропускает клики (`pointer-events: none`).

7. **Защита от двойного клика** — переменная `_tutTransitioning` блокирует повторные нажатия.
   Между шагами задержка 300ms.

8. **Прогресс** — градиентная полоска сверху + счётчик «N / Total» справа. Пользователь видит
   сколько шагов пройдено.

9. **Единый стиль для ВСЕХ шагов** — не только первых 5. Каждый шаг обязан иметь:
   icon, title, description. Опционально: selector, action, actionSelector.

10. **Позиционирование карточки** — ниже элемента (если есть место), иначе выше, иначе по центру.
    Автокоррекция overflow после рендера.

### Шаблон шага обучения (НОВЫЙ):
```javascript
{
    icon: '📊',                          // Эмодзи-иконка
    title: 'Короткий заголовок',         // Заголовок 21px bold
    description: 'Подробное описание',    // Текст 16px
    selector: '#statsGrid',              // CSS-селектор элемента (опционально)
    action: 'Нажмите на плитку...',      // Текст действия (опционально, делает шаг интерактивным)
    actionSelector: '.specific-btn'      // Альтернативный селектор для клика (опционально)
}
```

## Перезапуск сервера (localhost)

**Проблема:** `OSError: [Errno 48] Address already in use`

**Решение:**
```bash
lsof -ti:8080 | xargs kill -9; sleep 1; python3 server.py
```

## Архитектура проекта

- **Сервер:** `server.py` (Python, http.server + SQLite)
- **Фронтенд:** `templates/index.html` (single-page app, vanilla JS)
- **БД:** `task_manager.db` (SQLite)
- **Порт HTTP:** 8080
- **Порт HTTPS:** 8443
- **Хостинг:** Render (auto-deploy from git push)

---

## Структура «Категории» (kanban)

**Архитектура:** Отделы → Воронки (категории)

- Боковая панель: список отделов. `catsTreeState.selectedDeptId` — текущий отдел.
- «Все отделы» (`selectedDeptId = null`): сводка-плитки по отделам + общий список задач.
  Добавление воронок ЗАПРЕЩЕНО в режиме «Все отделы».
- Конкретный отдел: колонки-воронки. Первая колонка «Новая задача» — задачи без категории.
- Создание воронки: `showCategoryModal()` — только название + цвет. Отдел подставляется автоматически
  из `catsTreeState.selectedDeptId`. Родительская категория (parent_id) НЕ используется.
- Каждая воронка = категория с `department_id`.

## Подзадачи и обсуждение

- Подзадачи встроены в главную вкладку задачи (`td-subtasks-inline`), а не на отдельной вкладке.
- Кнопка 💬 рядом с подзадачей раскрывает **inline-чат** (`subtask-chat-panel`) прямо под строкой подзадачи.
- В чате: текстовые сообщения, @упоминания (`initMentionDropdown`), прикрепление файлов (до 8 МБ), голосовые сообщения.
- Повторный клик на 💬 закрывает чат. Открытие чата другой подзадачи закрывает предыдущий.
- Функции: `openSubtaskFull()`, `loadSubtaskChatComments()`, `sendSubtaskComment()`, `handleSubtaskChatFile()`, `toggleSubtaskChatVoice()`.
